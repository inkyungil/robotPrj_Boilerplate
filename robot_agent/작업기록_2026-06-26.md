## 추가 작업 — 0.02 재매핑 & 노이즈 튜닝 (오후)

### 경과
- navigation 재시작만으로는 20cm 틈이 안 열림 → 원인: **주행 맵이 아직 0.05** (`my_map.yaml`, 82×63).
  - 0.05에서 20cm 틈은 4칸뿐 → footprint inscribed(6cm)로 양쪽이 막혀 통과 불가.
  - 해상도는 새 맵 만들 때만 적용되므로 **재매핑 필수**임을 재확인.
- 사용자가 0.02로 재매핑 → `/home/pinky/map1.yaml` (resolution 0.020, 111×105, 약 2.2m×2.1m) 생성.
- 결과 맵 확인: **방 안쪽·벽·두 물체는 깨끗**, 단 **벽 바깥 잡음**(좌상단 부채꼴 + 하단 흩뿌림) 있음.

### 노이즈 원인
- 방은 ~2.2m인데 `max_laser_range: 10.0` → 문틈으로 라이다가 멀리 새어나가 벽 밖에 잡음 다수 생성.

### mapper_params.yaml 변경 (근본 해결, src+install 양쪽 적용)
| 항목 | 이전 → 변경 | 이유 |
|---|---|---|
| max_laser_range | 10.0 → **3.0** | 작은 방: 멀리 안 쏴서 벽 밖 잡음 제거 |
| minimum_travel_distance | 0.5 → **0.2** | 스캔 자주 모아 평균 → 선명 |
| minimum_travel_heading | 0.5 → **0.3** | 회전 중에도 자주 스캔 → 코너 깔끔 |
| scan_buffer_maximum_scan_distance | 8.0 → **3.0** | laser range와 일관성 |
- resolution 0.02 유지. 원본은 `mapper_params_20260626.yaml` 백업 있음.

### "청소 vs 재매핑" 결론
- 맵 청소(.pgm 직접 편집) = 결과 그림만 손봄 → **근본 아님**(매핑 때마다 재발).
- 채택: **B안(설정 튜닝)으로 처음부터 깨끗하게** + 필요시 마무리 청소.
